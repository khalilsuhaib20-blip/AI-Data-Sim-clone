Build a web application MVP called **"AI Data Company Simulator Portfolio"**.

The goal of the app is to simulate a real data team workflow where tasks are continuously generated by different roles in a company (stakeholders, software engineers, data engineers, data scientists, ML engineers, analysts).

The portfolio will show tasks that I complete as if I am working in a real company.

TECH STACK:

* Frontend: React with Tailwind
* Backend: Python FastAPI
* Database: SQLite (for MVP)
* LLM integration: OpenAI API (or placeholder function for now)

MAIN FEATURES:

1. Home Page
   Explain that this website simulates a real data company where tasks are generated by different roles and I complete them as part of my portfolio.

2. Task Board Page
   Create a Kanban-style board with three columns:

* Backlog
* In Progress
* Completed

Each task should contain:

* task_id
* title
* description
* role_that_requested_task (Stakeholder, Software Engineer, Data Engineer, Data Scientist, ML Engineer)
* priority
* project_area (Data Engineering, Data Science, Analytics, MLOps)
* status
* created_at

3. Task Details Page
   When clicking a task, show:

* full description
* role that requested it
* my solution notes
* GitHub link
* architecture notes
* completion status

4. Generate Task Button
   Create a button called "Generate New Task".

This should call a backend endpoint that asks the LLM to generate a realistic task request coming from one of these roles:

* Stakeholder
* Software Engineer
* Data Scientist
* Data Engineer
* ML Engineer

Prompt example for the LLM:

"You are simulating a real tech company. Generate a realistic task request for a data team member. The task should come from one of these roles: stakeholder, software engineer, data scientist, ML engineer, or analyst. Provide title, description, priority, and project area."

Return structured JSON.

5. Backend API

Create endpoints:

GET /tasks
POST /tasks
PATCH /tasks/{id}/status
POST /generate-task

6. Database Tables

tasks table:

id
title
description
requested_by
priority
project_area
status
created_at
solution_notes
github_link

7. UI Requirements

* Clean modern UI
* Use Tailwind
* Kanban style task board
* Task cards clickable
* Button to generate new tasks

8. Portfolio Goal

The app should feel like a continuous workflow of a real company where new tasks keep coming and I complete them over time.

Keep the MVP simple but structured so it can later support:

* multiple projects
* AI explanations of tasks
* project timelines
* documentation pages
